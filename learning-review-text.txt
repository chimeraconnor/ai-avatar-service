### 2. Automated Systems Need Explicit Criteria
System-event messages are just text — they don't carry algorithmic logic. I need to write explicit criteria like "priority: high → promote" that ANY agent can follow deterministically.

---

### 3. Resource Monitoring Prevents System Failures
Discord crashed with "ENOSPC: no space left on device." The filesystem was 100% full.
Resolution: Deleted moshi-tts (15GB) → freed ~14GB, disk went to 81% full
Lesson: Monitor disk usage with df -h. Clean up when approaching 90%, not after 100%.

---

### 4. Tool Selection Directly Impacts User Experience
Discord messages were taking 5-10 minutes. Log analysis revealed "API rate limit reached" errors for kimi-coding (k2p5).
Resolution: Switch Discord to use glm-4.7 (no rate limits, faster response)
Lesson: Log analysis reveals root causes. Fix the root cause, not the symptom.

---

### 5. Documentation Must Sync with Reality
FFmpeg was installed but TOOLS.md still said it was NOT available.
Lesson: When new tools are installed or removed, update documentation immediately.

---

### 6. QMD Setup Requires Persistent Paths in Docker
QMD broke after Docker restart. Installed bun to workspace (~/.openclaw/workspace/.bun/) for persistence.
Lesson: Use workspace paths, not system paths. They survive Docker restarts.

---

### 7. Browser Automation: DOM Refs Expire After Each Action
Wordle automation kept failing with "element not found" errors from stale DOM refs.
Lesson: Every click/type updates the page, so cached refs become invalid. Always take fresh snapshots.

---

### 8. GitHub CLI: PAT Tokens Are the Reliable Method
Device code flow kept timing out.
Resolution: Use gh auth login --with-token with PAT token

---

### 9. Timezone Conversion Is Mandatory for Communication
Lesson: Always convert UTC to IST when stating times. 7:00 PM UTC = 12:30 AM IST (next day).

---

### 10. Read Files Fully Before Editing
I mixed first and second person in SOUL.md.
Lesson: Read the FULL file before editing — note the voice/perspective, match existing style.
