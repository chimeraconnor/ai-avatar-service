**Evidence from logs:**

| 05:53:16 | 73.7 seconds | Slow listener |
| 05:54:44 | 273.9 seconds | Slow listener |
| 06:01:30 | 652.8 seconds (11 min!) | Slow listener |
| 06:40:46 | 261.8 seconds | Slow listener |
| 06:41:36 | 115.4 seconds | Slow listener |
| 07:10:43 | 337.9 seconds | Slow listener |

**Critical log entry (07:05:06 - 07:10:41):**
```
embedded run start: provider=kimi-coding model=k2p5 messageChannel=discord
embedded run agent end: isError=true error=⚠️ API rate limit reached. Please try again later.
Duration: 5.6 minutes (335,088 ms)
```

**Analysis:**
- Discord sessions using kimi-coding (k2p5) hit API rate limits
- After rate limit, messages wait 5-10 minutes before retry succeeds
- Some Discord messages also use zai/glm-4.7 (faster, no limits)

**Recommendation:** Configure Discord to use glm-4.7 instead of k2p5
- Eliminate rate limit issues
- Reduce response time from 5-10 minutes to 1-2 minutes
- Better Discord experience
