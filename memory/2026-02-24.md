# 2026-02-24 - Daily Notes

## Completed: Discord Voice Message Skill

**What we built:**
- Custom Python skill for sending native Discord voice messages with waveform visualization
- Works around OpenClaw Issue #16103 (broken native `asVoice` parameter)
- 3-step Discord API flow: Upload URL → CDN Upload → Voice Message Send

**Technical implementation:**
- Python script at `skills/discord-voice/scripts/send_voice.py`
- Auto-converts any audio to OGG/Opus using ffmpeg
- Generates 256-sample waveform from audio amplitude (base64 encoded)
- Auto-reads Discord token from OpenClaw config
- Per-step error handling with descriptive messages

**Repositories:**
- Main repo: Part of `chimeraconnor/anastasia` under `skills/discord-voice/`
- Standalone repo: `chimeraconnor/discord-voice-skill` (for easy discovery)

**Documentation updated:**
- `TOOLS.md` — Added precedence warning, TTS integration examples
- `SKILL.md` — Full manifest with "USE THIS, NOT native asVoice" section
- `README.md` — User-facing docs with comparison table

**Key lesson:** Native OpenClaw `asVoice` returns generic "Error" even when it works. This skill provides reliable feedback.

**Usage pattern:**
```bash
# Generate TTS audio
~/.openclaw/tools/tts-speak.sh "Hello Mr. Grey" /tmp/voice.wav kokoro 1

# Send as Discord voice message
python3 ~/.openclaw/workspace/skills/discord-voice/scripts/send_voice.py \
  --channel-id 1475566112019058758 \
  --audio-file /tmp/voice.wav
```

**Timing:**
- TTS generation: ~9-10 seconds (sherpa-onnx Kokoro)
- Python processing + Discord upload: ~1-2 seconds
- Total: ~10-12 seconds per voice message
